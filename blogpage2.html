    What does it mean to have a cryptocurrency?

    A cryptocurrency is a digital currency, which is an alternative form of payment created using blockchain

    But what is a Blockchain?

    The Wikipedia says,

    A blockchain is a type of distributed ledger technology (DLT) that consists of growing list of records, called blocks, that are securely linked together using cryptography.

    Well this doesn't help much, at least for a beginner.

    Let's take a bit more constructive approach towards the topic

     Why use a ledger?

    Say you have a group of few friends and you exchange money on various occasions throughout the month. It might be inconvenient to keep cash always to exchange, instead you create a website like a ledger where you keep entries like,

    Alice giver Bob 100 Rs

    Bob gives Jim 50 Rs.

    And so on...

    Now this ledger should be,

    * Public, so that during each transaction one can add lines to it

    * To confirm the transactions it should contain the signature of the payer.

    How to confirm transactions in a ledger?

    Now the problem is how to make a digital signature that will be infeasible?

    For this we will use a pair of strings(in computer terms) say keys.

    One will be the Private or Secret key(sk) which serves as a secret password that proves authenticity other will be the Public key(pk) that serves as a public name and address.

    Now to sign and verify the document we will create two functions,

    The Sign Function,

    This function will remember the payer and also the amount, so it has to be message dependent. It can be represented as follows,

    Sign (Message, SK) = Signature (Usually 256 bit binary code)

    The Verify Function,

    This function will verify the payer, the receiver and the amount.

    It can be represented as follows,

    Verify (Message, Signature, PK) = True/False [Depending on whatever the transaction is actually signed by the payer]

    The problem with central authority

    Although the transaction is secured there is still a big problem, who hosts the website of the ledger? Whoever hosts it will have the power to alter the rules and entries.

    The best solution is for everyone to keep their own copy of the actual ledger and update it with time as transactions happen.

    So the centralized ledger becomes a decentralized ledger.

    Order and arrangement of transactions

    To keep an order in the list of transactions there should be a timestamp on each transaction based on a unique order system for everyone so that at a given time it has the same value for everyone on every corner of earth.

    Now some of these transactions are stacked one by one in bundles, called Blocks.

    And the sequence of consecutive blocks is called a Blockchain.

    What if one receives contradicting blocks at once? Which one to trust?

    Well the bitcoin blockchain solves it by looking for proof-of-work.

    The block with the most proof-of-work is accepted into the ledger.

    What is proof-of-work?

    In the bitcoin network the members called miners confirm the transactions and add them to a block. Once a block is ready they try to find a number 'n' such that it when added to the end of the block and passed through a hashing function [A function that encrypts messages to binary stream, like SHA-256] creates a unique hash, say one starting with 30 zeros.

    There is no other way to do it but to iterate through all the possible numbers and hope to get it faster than other miners. The first of the miners to get the number gets their block accepted to the ledger and gets a certain prize money called mining reward much like a lottery system.

    Although that money too is a part of the ledger.

    This process is called proof-of-work mechanism.

    How proof-of-work confirms safety?

    Once a block is accepted immediately miners start working on the next block.

    If one happens to mine a corrupted block faster than all others and by sending faster than others adds it to the receiver's ledger still the honest miners block will reach the receiver. Now having two contradicting blocks the receiver will have to wait for next blocks. If the fraud sender doesn't keep mining and adding blocks to the chain faster than all other honest miners [Which is a absurdly impossible task] eventually his fraud chain will fall short and as per the bitcoin chain protocol,

    "The longest chain is always correct"

    So the fraud chain will be discarded eventually.

    Do we need all these details to trade or use cryptocurrencies on blockchains?

    No. But to stay safe from frauds and to delve deeper later on, this is all what one should know to the least.

    Cryptocurrencies are only a use case of blockchain technology.

    This is far from a comprehensive guide on the topic but this should be enough to spark interest in the regarding topic.
